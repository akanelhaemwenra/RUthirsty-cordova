# 📅🌤️ 日历和天气功能说明

## 新增功能概述

v1.0.4 版本新增了日历和天气显示功能，让用户可以：
- 查看当前日期和星期
- 实时显示当前天气状况和温度
- 根据天气情况获得智能喝水提醒

## 功能详情

### 1. 日历显示 📅

**位置**：应用顶部，标题下方左侧

**显示内容**：
- 当前日期（年月日）
- 当前星期

**特点**：
- 自动更新（每分钟刷新一次）
- 美观的卡片式设计
- 与应用整体风格一致的渐变效果

**示例显示**：
```
📅 2024年2月7日
   星期三
```

### 2. 天气显示 🌤️

**位置**：应用顶部，标题下方右侧

**显示内容**：
- 当前温度（摄氏度）
- 天气状况描述（晴、多云、雨等）
- 天气图标（emoji）

**特点**：
- 基于地理位置的实时天气
- 智能缓存（30分钟更新一次）
- 自动定位或使用默认城市
- 根据天气调整喝水提醒

**天气图标对照**：
- ☀️ 晴天
- ⛅ 多云
- ☁️ 阴天
- 🌧️ 雨天
- ⛈️ 雷雨
- ❄️ 下雪
- 🌫️ 雾霾
- 🌙 夜间晴朗

### 3. 智能天气提醒 💡

应用会根据天气情况自动调整激励消息：

| 天气条件 | 提醒消息 |
|---------|---------|
| 温度 > 30°C | "天气炎热，记得多喝水补充水分！🔥" |
| 温度 > 25°C | "温暖的天气，保持水分很重要！☀️" |
| 温度 < 10°C | "天气寒冷，温水更适合哦！❄️" |
| 下雨天 | "下雨天也要记得喝水哦！🌧️" |
| 晴天 | "阳光明媚，多喝水保持活力！☀️" |

## 配置天气 API

### 获取免费 API 密钥

1. **访问 OpenWeatherMap**
   - 网址：https://openweathermap.org/api
   - 点击 "Sign Up" 注册账号

2. **获取 API Key**
   - 登录后进入 "API keys" 页面
   - 复制默认的 API key 或创建新的
   - 免费账户限制：60次/分钟，1,000,000次/月

3. **配置到应用**
   - 打开 `www/js/index.js`
   - 找到两处 `YOUR_API_KEY_HERE`
   - 替换为你的 API key

**示例**：
```javascript
// 第 548 行附近
var apiKey = 'YOUR_API_KEY_HERE'; // 替换为：'abc123def456...'

// 第 556 行附近
var apiKey = 'YOUR_API_KEY_HERE'; // 替换为：'abc123def456...'
```

### 不使用 API 的情况

如果不配置 API key，天气功能会显示：
- 图标：🌤️
- 温度：--°C
- 描述：天气信息不可用

应用的其他功能不受影响。

## 技术实现

### 日历功能

```javascript
// 自动更新日期显示
updateCalendar: function() {
    var now = new Date();
    var weekdays = ['星期日', '星期一', '星期二', '星期三', '星期四', '星期五', '星期六'];

    var year = now.getFullYear();
    var month = now.getMonth() + 1;
    var day = now.getDate();
    var weekday = weekdays[now.getDay()];

    // 更新显示
    dateElement.textContent = year + '年' + month + '月' + day + '日';
    weekdayElement.textContent = weekday;
}
```

### 天气功能

**工作流程**：
1. 尝试获取用户地理位置
2. 如果成功，使用经纬度获取天气
3. 如果失败，使用默认城市（北京）
4. 缓存天气数据30分钟
5. 每30分钟自动更新一次

**地理定位**：
```javascript
navigator.geolocation.getCurrentPosition(
    successCallback,
    errorCallback,
    { timeout: 10000, enableHighAccuracy: false }
);
```

**API 调用**：
```javascript
// 使用经纬度
https://api.openweathermap.org/data/2.5/weather?lat={lat}&lon={lon}&units=metric&lang=zh_cn&appid={apiKey}

// 使用城市名
https://api.openweathermap.org/data/2.5/weather?q={city}&units=metric&lang=zh_cn&appid={apiKey}
```

**缓存机制**：
```javascript
// 保存到 localStorage
localStorage.setItem('weatherCache', JSON.stringify(data));
localStorage.setItem('weatherCacheTime', now.toString());

// 检查缓存（30分钟有效）
if (cachedWeather && (now - cacheTime) < 1800000) {
    // 使用缓存数据
}
```

## 权限配置

### Content Security Policy (CSP)

已更新 `index.html` 的 CSP 策略以允许天气 API 访问：

```html
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self' data: https://ssl.gstatic.com https://api.openweathermap.org 'unsafe-eval' 'unsafe-inline';
               style-src 'self' 'unsafe-inline';
               media-src *;
               img-src 'self' data: content: https:;">
```

### Cordova 权限

如需地理定位功能，确保 `config.xml` 包含：

```xml
<plugin name="cordova-plugin-geolocation" source="npm" />
```

在 Android 上还需要：
```xml
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.INTERNET" />
```

## 故障排除

### 问题 1：天气显示 "天气信息不可用"

**可能原因**：
- 未配置 API key
- API key 无效
- 网络连接问题
- API 请求限制

**解决方案**：
1. 检查 API key 是否正确配置
2. 验证 API key 是否有效（登录 OpenWeatherMap 查看）
3. 检查网络连接
4. 等待一段时间（可能达到请求限制）

### 问题 2：地理定位失败

**可能原因**：
- 用户拒绝位置权限
- 设备不支持地理定位
- 定位超时

**解决方案**：
- 应用会自动使用默认城市（北京）
- 可以修改代码中的默认城市

### 问题 3：日期显示不正确

**可能原因**：
- 设备时间设置错误

**解决方案**：
- 检查设备系统时间设置
- 确保时区设置正确

## 开发者注意事项

### API Key 安全

⚠️ **重要**：不要将 API key 提交到公共代码仓库

**最佳实践**：
1. 使用环境变量
2. 在 `.gitignore` 中排除配置文件
3. 使用构建时替换

**示例**：
```javascript
// 开发环境
var apiKey = process.env.WEATHER_API_KEY;

// 或使用配置文件
var config = require('./config.json');
var apiKey = config.weatherApiKey;
```
